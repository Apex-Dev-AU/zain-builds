## What this is

This is me learning how to actually build and deploy websites with AI.

Not just generate code.

Actual websites.  
Files.  
Folders.  
GitHub repos.  
Vercel deployments.  
Domains.  
Errors.  
Fixes.  
Updates.

The interesting part for me is not that AI can write code.

It is that I can now take an idea further than I could before and actually get something live.

## What I’m really learning

- how websites are actually structured
- how to work inside VS Code
- how GitHub fits into the workflow
- how Vercel deploys from a repo
- what “production” actually means
- how to trace and fix build errors
- how to update something and push changes properly

Before this, all of that felt separate from me.

Now it feels a lot more practical.

### VS Code

Once I opened the folders and files enough times, it stopped feeling abstract.

## Main thing I’ve realised

AI helps a lot.

But it still needs direction.

If the direction is vague, the output is vague.  
If the idea is clear, AI speeds everything up.

So the real skill is not just using AI.

It is:
- knowing what you want
- explaining it properly
- spotting what is off
- adjusting it
- pushing it live
- fixing what breaks
