# Documentation

This folder contains everything you (future-you) need to operate
this site without re-figuring it out. Read in this order if you're
new to the codebase, or coming back after a long absence.

1. **[ARCHITECTURE.md](./ARCHITECTURE.md)** — How the site is
   structured. The big picture.
2. **[CONTENT.md](./CONTENT.md)** — How to write notes and
   projects. Frontmatter contracts.
3. **[STYLE.md](./STYLE.md)** — Voice and writing conventions.
   Read before every writing session for the first six months.
4. **[DESIGN.md](./DESIGN.md)** — The design system. Tokens,
   typography, color, spacing.
5. **[WORKFLOWS.md](./WORKFLOWS.md)** — How to publish a note,
   ship a project, deploy a change.
6. **[DECISIONS/](./DECISIONS/)** — Architecture Decision
   Records. One per significant choice. Never delete; supersede.

## Conventions

- Documentation is part of the product. Update it when behaviour
  changes.
- Plain language. No jargon you wouldn't say out loud.
- Examples over abstractions.

## When something feels off

Read the relevant doc here first. If the doc disagrees with the
code, the doc is wrong — fix the doc. If you're tempted to add a
new document, ask whether it belongs in an existing one.
