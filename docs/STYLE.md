# Style

The voice of this site is **considered, plainspoken,
in-progress**. Open this file at the start of every writing
session for the first six months. After that you'll know it
by heart.

## Core principles

1. **Write for a curious adult.** Not a beginner, not a
   specialist. Someone smart who hasn't thought about this
   thing as much as you have.
2. **First person, no performance.** "I tried," "I learned,"
   "I was wrong." Not "you should," "everyone knows,"
   "obviously."
3. **Specific over abstract.** Real examples. Real numbers.
   Real names. "The car has 90,000km" beats "the car has
   high mileage."
4. **Plain words over fancy ones.** "Use" not "utilise."
   "Help" not "facilitate." "About" not "regarding."
5. **Sentence case for headings.** Not Title Case. Not
   ALL CAPS.
6. **Avoid the second person when not addressing the reader.**
   "When you build a CMS" is usually weaker than "When I
   built a CMS."

## Sentence-level rules

- **Short sentences.** Then a longer one for rhythm.
- **One thought per paragraph.** If a paragraph has two
  ideas, split it.
- **No exclamation points** unless something is genuinely
  exclamatory. One per ten thousand words.
- **No emojis** in body content. They date the writing and
  don't translate to print or RSS readers.
- **No throat-clearing.** Don't start with "In this post..."
  or "I've been thinking a lot about..." Just start.
- **Honesty over confidence.** "I think," "I'm not sure
  yet," "this might be wrong" are signs of life.

## Words to avoid

These appear in writing that hasn't been edited. If you
catch yourself reaching for them, rewrite the sentence:

| Avoid | Prefer |
|---|---|
| utilise, leverage | use |
| facilitate, enable | help, let |
| in order to | to |
| at this point in time | now |
| due to the fact that | because |
| in the event that | if |
| a number of | some, several, [number] |
| myself / yourself (used wrong) | me / you |
| journey | (just delete it) |
| dive deep | go into detail |
| game-changer, revolutionary | (delete) |
| literally (when figurative) | (delete) |

## Punctuation

- **Em dashes** for asides — like this one — set with no
  space, or with a thin space if your tool supports it.
- **En dashes** for ranges: `2017–2026`, `pages 12–15`.
- **Smart quotes** ('curly' and "curly"), not straight
  quotes. Markdown handles this if you let it.
- **Australian English** spelling: colour, organise,
  centre, programme (for a TV programme), program (for
  software).
- **Oxford comma** when it adds clarity, optional when it
  doesn't.

## Titles

- Specific. "What I learned shipping the Tiguan downpipe"
  beats "Lessons learned."
- Sentence case.
- No clickbait. No "you won't believe."
- One topic per post. If the title has "and" in it, the
  post probably wants to be two posts.

## Summaries (frontmatter)

- One sentence. Two at most.
- Treat it as the OG card and the RSS preview — because it
  is.
- Don't repeat the title. Add to it.

## What to publish

The bar isn't "this is finished and polished." The bar is
"this is honest, accurate, and worth someone's time."

Working notes are welcome. Half-baked ideas are welcome. Tag
them `note` or `log` instead of `essay` and ship them.

The only thing not welcome is filler.
